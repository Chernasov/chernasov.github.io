Курсовой проект к модулю "Автоматизированное тестирование"
закрепление полученных заданий по использованию инструментов автоматизации, паттернов в процессе автоматизации тестирования комплексного сервиса, взаимодействующего с СУБД и API Банка

Ключевая задача - автоматизировать сценарии (как позитивные, так и негативные) покупки тура. Основные критерии:

  • Автоматизировать тесты для двух форм покупки тура
  • Обязательно должны быть тесты UI
  • Обязательно должны быть репорты
  • Обязательно должны быть запросы в базу, проверяющие корректность внесения приложением информации
  • Автоматизировать API тесты для приложения.
  • Реализовать возможность запуска в контейнерах двух баз данных (Mysql и Postgres) и параметризировать запуск приложения и тестов, передавая в параметрах разные конфиги для подключения к любой из запущенных баз данных

Использование

  • паттерн Page Object – для более гибкого взаимодействия с SUT и сокрытия внутреннего устройства определённой страницы или её виджетов за логическим интерфейсом взаимодействия
  • REST-assured – для тестирования API
  • Selenide – для тестирования GUI
  • Apache Commons – для запроса к БД
  • Docker - для развертывания БД

для тестирования веб-сервиса. Проект расположен на GitHub

Создана коллекция для CRUD-операций, в коллекции используются переменные для токена, имени файла, отбора Id созданного юзера.

После создания юзера выполняется скрипт для присвоения переменной "user_id" id созданного юзера.

При выполнении дальнейших операций по чтению, изменению и удалению юзера используется переменная.

Также при создании и изменении юзера для поля "email" используется генерация случайных значений.
